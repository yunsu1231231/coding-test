### 1. 틀린이유
1. 0 based vs 1 based  
대원칙: 문제가 번호룰 주면 -> 1 based, 그렇지 않으면 0 based

### 2. 대원칙
- 문제에서 원본 데이터를 0-based로 줬다면 그대로 0-based  
- 문제에서 번호(1~N)를 줬다면 1-based  
- 표 기반 DP(LCS/편집거리)는 문제 구조상 dp는 1-based여야 함  
- 그래서 문자열 비교 DP는 “문자열 0-based + DP 1-based” 조합이 생김

(문제가 그렇게 두 개를 따로 정의해버렸기 때문)

결론: 배열이든 dp든 문제에서 주어진 조건에 맞춰서 based를 따르는 게 정답이고
문제에서 두 개의 based가 다르게 주어지면
원본은 원본대로, dp는 dp대로 각각 based를 유지하는 것이 일관성 있는 해석하기
